Validacin cruzada de regresin logstica con seleccin de variables

Para reducir la dimensionalidad del dataset (155 variables) y trabajar solo con las ms relevantes, se aplic un mtodo de seleccin univariada basado en la estadstica ANOVA F (f_classif).

Este enfoque mide, para cada variable independiente, cunto vara su media entre las clases de la variable de salida (dura_leg_bin). En otras palabras, el F-statistic compara la variabilidad entre grupos (clases 0 y 1) contra la variabilidad interna dentro de cada grupo.

  • Valores altos de F indican que la variable tiene un mayor poder discriminante entre las clases.
  • p-valores bajos (cercanos a 0) confirman que la variable est significativamente asociada con la variable objetivo.

Con las probabilidades Prob_Clase_1 del modelo, se generaron predicciones para tres umbrales de decisin: 0.3, 0.5 (estndar) y 0.7. Para cada umbral se construy la matriz de confusin y se calcularon:

import matplotlib.pyplot as plt
from sklearn.metrics import ConfusionMatrixDisplay
from sklearn.metrics import confusion_matrix, accuracy_score, recall_score

# Definir umbrales
thresholds = [0.3, 0.5, 0.7]

for thr in thresholds:
    print(f"\n=== Resultados con umbral = {thr} ===")
    
    # Generar predicciones segn el umbral
    preds_thr = (results["Prob_Clase_1"] >= thr).astype(int)
    
    # Matriz de confusin
    cm = confusion_matrix(y_test, preds_thr)
    disp = ConfusionMatrixDisplay(cm, display_labels=model.classes_)
    disp.plot(xticks_rotation="vertical")
    plt.show()
    
    # Calcular mtricas
    acc = accuracy_score(y_test, preds_thr)
    sens = recall_score(y_test, preds_thr, pos_label=1)   # sensibilidad (TPR)
    espec = recall_score(y_test, preds_thr, pos_label=0)  # especificidad (TNR)
    
    print(f"Exactitud:     {acc:.3f}")
    print(f"Sensibilidad:  {sens:.3f}")
    print(f"Especificidad: {espec:.3f}")

Cuando el umbral se fij en 0.3, el modelo se volvi ms permisivo para declarar que un caso perteneca a la clase 1. Esto provoc que la sensibilidad subiera hasta 0.918, es decir, se recuper a casi todos los verdaderos positivos. Sin embargo, la contracara fue que la especificidad descendi a 0.731, ya que el modelo comenz a confundir con mayor frecuencia a los negativos como positivos, produciendo ms falsos positivos. Esta configuracin es adecuada si lo ms importante es no dejar escapar matrimonios con duracin por encima del promedio, aunque ello implique marcar algunos casos de forma incorrecta.

En cambio, con un umbral ms estricto de 0.7 la dinmica se invierte. La especificidad se elev hasta 0.923, lo que significa que casi no se clasificaron falsamente como 1 los casos que en realidad eran 0. Sin embargo, la sensibilidad se redujo drsticamente a 0.694, reflejando que el modelo dej pasar muchos positivos reales sin detectarlos (falsos negativos). Esta situacin resulta til cuando es ms grave etiquetar incorrectamente a un 0 como 1 que dejar escapar a varios 1.

Finalmente, con el umbral estndar de 0.5, el modelo alcanz el mejor balance entre ambas mtricas. La sensibilidad fue de 0.824 y la especificidad de 0.854, lo que a su vez se reflej en la exactitud ms alta de los tres escenarios (0.841). En este punto se observa un equilibrio: el modelo no es excesivamente permisivo ni demasiado estricto, y mantiene un rendimiento aceptable en ambas direcciones.

Curva ROC y evaluacin del rea bajo la curva (AUC)

A partir de las probabilidades de clase 1 calculadas por la regresin logstica, se construy la curva ROC (Receiver Operating Characteristic). Esta curva grafica la relacin entre la tasa de verdaderos positivos (sensibilidad) y la tasa de falsos positivos (1 especificidad) a lo largo de todos los posibles valores de umbral de decisin. Adems, se calcul el rea bajo la curva (AUC), que resume en un solo valor la capacidad discriminativa del modelo.

In[86]:
